web, html

Those links you often click are not regular text. These are called `hypertext` and they allow you to surf the internet. Without them, there would be no surfing. We would consume websites similarly to how we read a physical book. The internet navigation would be solely based on using specific URLs that would be inserted into a browser. The browser would request and showcase the static data and you would consume it.

`Hypertext` is composed of two words `hyper` meaning beyond and `text` well text. So if you have heard about this term you now know that is basically text that has beyond capabilities allowing everyone NOT to read in a fixed order. Making surfing the internet possible, exciting and ending up in places you never thought. Isn't that cool? There is so much that we give for granted and never stop to admire.

I'm now going to use a hypertext so that you can look into the person who envisioned it in the early 1960's. And that person was [Ted Nelson](https://en.wikipedia.org/wiki/Ted_Nelson), if you already didn't know about him.

In HTML for sure you've used the `href` attribute, an abbreviation of `Hypertext Reference`, which is used to specify a `hyperlink` destination. This attribute can be used within different HTML elements for different purposes:

```html
<!-- anchor -->
<!-- common hypertext element used for links -->
<a href="https://dev.to">dev community</a>

<!-- external resource link -->
<!-- used inside <head> to link resources like stylesheets, icons, etc. -->
<link rel="stylesheet" href="styles.css" />

<!-- area -->
<!-- defines clickable regions inside an image map -->
<area shape="rect" coords="0,0,100,100" href="page.html" />

<!-- base -->
<!-- defines the base URL for relative URLs in the document. -->
<base href="https://example.com/" />
```

In later years the web itself was originally envisioned as a giant `hypertext system` by [Tim Berners-Lee](https://en.wikipedia.org/wiki/Tim_Berners-Lee) while working at CERN. The idea was simple but powerful: documents connected to other documents through references, allowing knowledge to be explored rather than read in a strict linear order.
